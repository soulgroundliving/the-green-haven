# กฏระเบียบ เงินมัดจำ · ค่าสัตว์เลี้ยง · ค่าเสียหาย — Nature Haven

> ร่าง 2026-06-04 จาก spec ที่เจ้าของยืนยัน · **ตรวจทานถ้อยคำก่อนใช้จริง** (ส่วนที่เป็นข้อผูกพันทางกฎหมายควรให้ผู้เชี่ยวชาญตรวจ) · ตัวเลข/กติกาทั้งหมดมาจากเจ้าของ ผมจัดเรียงให้ชัดเท่านั้น

---

## ส่วนที่ 1 — กฏระเบียบ (สำหรับผู้เช่า)

### 1.1 เงินมัดจำห้องพัก
- เงินมัดจำ = **ค่าเช่า 2 เดือน** เก็บรักษาไว้ คืนเมื่อสิ้นสุดสัญญา
  - ห้อง 1,500 → มัดจำ **3,000** · ห้อง 7,000 → มัดจำ **14,000**
- **ผ่อนชำระมัดจำได้** — ส่วนที่ยังไม่ครบถือเป็นยอดค้างชำระจนกว่าจะครบ
- เงินมัดจำเป็น **เงินคืนได้** ไม่ใช่ค่าธรรมเนียม

### 1.2 ห้องเลี้ยงสัตว์ (เฉพาะห้องที่อนุญาต)
- **ค่าธรรมเนียมสัตว์เลี้ยง 400 บาท/ตัว/เดือน** — เรียกเก็บรวมในบิลรายเดือน
- **เงินประกันสัตว์เลี้ยง 10,000 บาท/ห้อง** — แยกต่างหากจากมัดจำห้อง
  - คืนได้ **เต็มจำนวน** หากไม่มีความเสียหายที่เกิดจากสัตว์เลี้ยง
  - มีไว้เพราะสัตว์เลี้ยงมีความเสี่ยงเฉพาะ (กลิ่น · ขน · การกัดแทะ · การทำความสะอาดลึก) — แยกออกมาเพื่อ **ปกป้องเงินมัดจำห้องของผู้เช่าเอง** ด้วย

### 1.3 การคืนเงินเมื่อย้ายออก
1. เดือนสุดท้ายออกบิลตามปกติ (ค่าเช่า + น้ำ + ไฟ + ค่าสัตว์เลี้ยงถ้ามี)
2. หักบิลเดือนสุดท้าย + ค่าเสียหาย (ถ้ามี) ออกจากเงินมัดจำ
3. คืนส่วนต่างให้ผู้เช่า พร้อมหลักฐานการโอน
   - **ตัวอย่าง:** มัดจำ 3,000 − บิลเดือนสุดท้าย 2,300 = **คืน 700 บาท**

### 1.4 ค่าเสียหาย
- **ความเสียหายจากผู้เช่า** → หักจากเงินมัดจำห้อง
- **ความเสียหายจากสัตว์เลี้ยง** → หักจากเงินประกันสัตว์เลี้ยงก่อน (หากเกิน หักจากมัดจำห้องต่อ)
- **ค่าปรับจุดที่เสียหาย:** เฉพาะอาคาร **Nest** จุดละ **500 บาท**
- **ความเสียหายร้ายแรง:** คิดตามค่าซ่อม/ค่าทดแทนจริง
- ทุกการหักมี **รูปถ่ายและรายละเอียดประกอบ** (โปร่งใส ตรวจสอบได้)
- หากค่าเสียหายเกินเงินที่วางไว้ ผู้เช่าชำระส่วนเกินเพิ่ม

### 1.5 หลักการ
ดูแลห้องและสัตว์เลี้ยงให้ดี = ได้รับเงินมัดจำและประกันคืนเต็มจำนวน

## ส่วนที่ 2 — สเปกระบบ (ภายใน — สำหรับสร้างฟีเจอร์)

### 2.1 การจัดประเภทบัญชี (blueprint น.1 "แยกประเภทรายได้")
| ประเภท | รายการ | บันทึกที่ |
|---|---|---|
| **หนี้สิน** (ไม่ใช่รายได้ · คืนได้) | มัดจำห้อง (2×ค่าเช่า) · ประกันสัตว์ (10,000) | `deposits/` |
| **รายได้** | ค่าเช่า · น้ำ · ไฟ · **ค่าสัตว์ 400/ตัว** · ค่าปรับเสียหาย | บิลรายเดือน / settlement |

### 2.2 การหักค่าเสียหาย (routing + overflow)
| ต้นเหตุ | ลำดับการหัก |
|---|---|
| **คน/ผู้เช่า** | มัดจำห้อง → (เกิน) บิลค้าง · **ไม่แตะประกันสัตว์** |
| **สัตว์** | ประกันสัตว์ → (เกิน) มัดจำห้อง → (เกิน) บิลค้าง |
| **กำกวม** | default = มัดจำห้อง + รูปถ่ายประกอบ |

*ความไม่สมมาตรตั้งใจ:* สัตว์ทำเกินล้นเข้ามัดจำห้องได้ (เงินผู้เช่าคนเดียวกัน) · คนทำห้ามกินประกันสัตว์ (ต้องคืนถ้าสัตว์ไม่ได้ทำ)

### 2.3 โครงสร้างข้อมูล (ต้อง verify ของจริงก่อน — Explore กำลังเช็ค)
- `deposits/{b}_{r}` — มัดจำห้อง = 2×ค่าเช่า · **ผ่อนจ่าย** (paidSoFar/due) · มี/เพิ่ม pet deposit แยก
- บิลรายเดือน — เพิ่ม field **`petFee` = 400 × จำนวนสัตว์** (ปิด #243) + จัดหมวดใน `aggregateMonthlyRevenue`
- **move-out settlement** — รายการหักย่อย `{type, cause(human/pet), desc, amount, photo}` → คืน/ค้าง + audit `DEPOSIT_RETURNED` + บันทึกสลิปโอนคืน
- shortfall (เกินมัดจำ) → บิลค้าง (ผูกกับ arrears/aging — Phase 2 ตัวสุดท้าย)

### 2.4 จุดเซฟ (safety guards — ห้องบิ้วแพง, ไม่มีเพดาน)
1. **รูปถ่าย + โน้ต ต่อรายการหัก = บังคับ** (เก็บแบบ PDPA-safe เหมือน checklist) — กันข้อพิพาท
2. **preview → ยืนยันชัด → audit row** (ไม่หักอัตโนมัติ §7-I)
3. **เกินมัดจำ → ออกบิลค้าง** (ไม่ตัดทิ้งที่ 0)
4. **บันทึกการโอนคืน + สลิป** (ปิด loop, กันเถียง "ไม่ได้คืน")
5. เตือนเมื่อยอดหักสูงผิดปกติ (fat-finger guard)

### 2.5 ผูกกับงานที่มี
- ปิด **#243** (pet fee — revenue category ที่เลื่อนเพราะ "ไม่มี field")
- ผูก **arrears/aging** (Phase 2 ตัวสุดท้าย — ผ่อนมัดจำ + shortfall = ยอดค้าง)
- **คนละ flow กับ refundBill** (เพิ่ง ship — คืนเงินบิลค่าเช่าที่จ่ายผิด ≠ คืนมัดจำย้ายออก)
